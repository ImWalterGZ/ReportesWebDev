What is it?

El desarrollo web es compuesto por un amplio compendio de herramientas donde cada una de ellas tiene un objetivo claro y una aplicación concreta.

Para la correcta gestión de los conceptos, estos se dividen en categorías que determinan el campo de acción de donde son aplicadas: Front-End y Back-End technologies.

Front-end technologies

Hypertext Markup Language, (Lenguage de marcado de hiper texto) es el codigo que sostiene la estructura basica de una pagina web, asegurandonos de contar con un formato correcto.

Cascade Style Sheets, determina el estilo y estética de una pagina web, es el lenguaje que acompaña a HTML para estilizarlo y describe como los elementos de un codigo HTML deben de ser mostrados.

Es el componente de las paginas web que permite la interactivdad y un comportamiento dinamico de la pagina, mejora la experiencia de usuario al permitirle interactuar directamente con el contenido.

Preprocesadores de CSS que permiten escribir estilos de manera más eficiente.

Son colecciones de códigos pre fabricados y herramientas diseñadas para formar un ambiente de desarrollo más completo y cómodo para el desarrollador, promoviendo mejores practicas para el desarrollo de UI y UX.

Los frameworks mas usados en la comunidad de desarrollo son React, angular, Vue.js, Svelte, Tailwind.

Framework de CSS que facilita el diseño responsivo y la creación de interfaces web.

Back-end technologies

El backend de una aplicación o pagina web se refiere a el código del lado del servidor (server-side code) que corre en el mismo.

Es responsable de procesar las peticiones del cliente, ejecutando procesos logicos prestablecidos y devolviendo la información apropiada al cliente, los componentes incluyen desde el servidor, el codigo, la aplicacion y la base de datos.

Server

El trabajo principal de un servidor de internet: Alojar, procesar y entregar paginas web a usuarios.

Un servidor sigue los siguientes pasos:

  1. Una persona ingresa una direccion URL en su navegador.
  1. El navegador obtiene la direccion IP del nombre del dominio, ya sea por traduccion de URL a través de DNS o accediendo a la memoria caché.
  1. El navegador pide el archivo de la pagina a el servidor que la aloja por medio de una HTTP request.
  1. El servidor envía la pagina solicitada por medio del protocolo mencionado.

Los servidores hacen que estas paginas puedan existir en todas partes al mismo tiempo, siendo un pilar en la gigante industria de internet.

Base de datos

Software que almacena y trabaja con los datos de websites o aplicaciones web. La base de datos es la ubicación central donde nuestra pagina y los datos que manejamos en esta viven, guarda datos de usuario, datos del negocio, datos de la lógica, todo esto a gran escala y de forma segura.

Un claro entendimiento de su funcionamiento permite que los desarrolladores web organicen datos en estructuras lógicas y puedan acceder a ellas de manera eficiente cuando sea necesario.
